# Arena Slaps — Sound Assets

This directory holds every `.ogg` sound effect shipped with the game. The
pipeline is **manifest-driven**: code never hardcodes filenames. The single
source of truth for which sounds exist, where they live on disk, and what
default volume they play at is
[`src/game/assets/soundManifest.ts`](../../src/game/assets/soundManifest.ts).

## How to replace a sound

Drop a new `.ogg` file into this directory using the **exact same filename**
as the entry in `soundManifest.ts`. No code change is required.

For example, to swap out the slap hit sound:

1. Export your new sound effect as `slap-hit.ogg` (Vorbis-encoded).
2. Overwrite `public/sounds/slap-hit.ogg`.
3. Refresh the page — `PreloadScene` reads the manifest verbatim and
   `Phaser.Audio.SoundManager` will load the new file. No rebuild needed.

To change the **path** or the **default volume** for a sound, edit the
matching entry in `SOUND_MANIFEST` inside
`src/game/assets/soundManifest.ts` — the per-sound `volume` field is a
multiplier on top of the global master volume (set in the menu).

## How to add a new sound

1. Add a new string literal to the `SOUND_KEYS` union in
   `soundManifest.ts` (this gives you a new `SoundKey`).
2. Add a matching entry to the `SOUND_MANIFEST` array (with `key`, `path`,
   and optional `volume`).
3. Drop the `.ogg` file into this directory using the filename you used in
   the `path` field.
4. (Optional but recommended) Add a high-level method on `AudioService`
   (e.g. `playFoo(): boolean { return this.play("foo"); }`) so callers don't
   have to remember the string key. Existing high-level methods follow the
   pattern `playSlapHit()`, `playMenuClick()`, etc.
5. (Optional) Add a unit test in `AudioService.test.ts` that asserts the new
   helper plays the correct key.

`PreloadScene` automatically iterates `SOUND_MANIFEST` and calls
`load.audio(key, path)` for every entry, so the new sound is loaded with
zero additional wiring.

## How to swap the audio engine entirely

The audio stack is split into two layers:

- `IAudioBackend` (`src/game/audio/AudioBackend.ts`) — the low-level
  contract. The default implementations are `PhaserAudioBackend`
  (production) and `NoopAudioBackend` (tests / SSR / browsers without
  WebAudio).
- `AudioService` (`src/game/audio/AudioService.ts`) — the high-level API
  that callers (scenes) actually use. It owns mute + master-volume scaling
  and delegates the actual playback to whatever `IAudioBackend` it was
  constructed with.

To swap the engine (e.g. to raw Web Audio API, Howler, or a stub for
testing):

1. Implement the `IAudioBackend` interface (methods: `load`, `isLoaded`,
   `play`, `stopAll`).
2. Pass an instance of your backend to `new AudioService(backend, settings)`.
   Scenes get their `AudioService` from `createAudioService()` in
   `src/game/audio/createAudioService.ts` — you can swap the factory's
   branch logic to return your backend instead of `PhaserAudioBackend`.

The manifest is engine-agnostic, so the same set of `.ogg` files and
`SoundKey`s work for any backend that supports playing audio buffers by
string key.

## Regenerating placeholders

If you delete or rename a placeholder and need to recreate it:

```bash
cd public/sounds
./generate-placeholders.sh
```

This requires `ffmpeg` on your PATH. Each tone is generated with a distinct
frequency (see the script) so you can audibly verify which key fires during
development. **These placeholders are not meant for production** — replace
them with real SFX before shipping.
